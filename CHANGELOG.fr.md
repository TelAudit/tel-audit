# Changelog

## v2.3 beta 3

Édition code lisible humain et améliorations des renvois d'appels.

- Ajout d'une documentation développeur simplifiée.
- Ajout de notes d'architecture simplifiées.
- Ajout de commentaires explicatifs dans les fichiers principaux et les règles d'audit.
- Amélioration de la détection des renvois actifs pour les sauvegardes Xorcom / Ombutel.
- Détection des renvois depuis `ombu_diversions` et `ombu_forwarding_diversions` lorsque ces données sont présentes.
- Les renvois désactivés avec ancienne destination conservée ne sont pas affichés comme actifs.
- L'audit automatique reste visible dans les rapports PDF et HTML même lorsqu'aucun problème n'est détecté.
- Ajout d'un message clair lorsque les règles d'audit actuelles ne détectent aucun problème.
- Aucun changement fonctionnel volontaire dans l'édition code lisible humain, en dehors des corrections bêta documentées.

## v2.3 beta

- Ajout d'une section HTML et d'un export CSV dédié aux renvois actifs.
- Ajout d'un export Draw.io dédié aux renvois actifs lorsqu'ils sont détectés.
- Préparation du modèle pour afficher les conditions associées lorsque la sauvegarde les fournit.
